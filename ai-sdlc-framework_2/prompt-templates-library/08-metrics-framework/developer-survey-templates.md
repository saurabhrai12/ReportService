# Developer Survey Templates for AI Adoption

## Overview

This document contains survey templates to measure developer satisfaction, AI tool effectiveness, and adoption progress. Use these surveys at regular intervals to track sentiment and gather qualitative feedback.

---

## Survey Analysis Guide

### Scoring Methodology

```python
# Survey response scoring
SATISFACTION_SCORES = {
    1: 'Very Dissatisfied',
    2: 'Dissatisfied', 
    3: 'Somewhat Dissatisfied',
    4: 'Neutral',
    5: 'Somewhat Satisfied',
    6: 'Satisfied',
    7: 'Very Satisfied',
    8: 'Highly Satisfied',
    9: 'Extremely Satisfied',
    10: 'Perfectly Satisfied'
}

# NPS calculation
def calculate_nps(scores):
    """Calculate Net Promoter Score from satisfaction ratings."""
    promoters = sum(1 for s in scores if s >= 9)
    detractors = sum(1 for s in scores if s <= 6)
    total = len(scores)
    return ((promoters - detractors) / total) * 100

# Satisfaction index
def satisfaction_index(scores):
    """Calculate satisfaction index (0-100)."""
    return (sum(scores) / (len(scores) * 10)) * 100
```

### Key Metrics to Track

| Metric | Calculation | Target | Alert Threshold |
|--------|-------------|--------|-----------------|
| Overall Satisfaction | Average of Q4 | ≥7.0 | <6.0 |
| NPS Score | Promoters - Detractors | ≥30 | <0 |
| Adoption Rate | % using AI regularly | ≥75% | <50% |
| Time Saved | Average hours/week | ≥5 hrs | <2 hrs |
| Quality Impact | % reporting improvement | ≥60% | <40% |

### Reporting Cadence

| Survey Type | Analysis | Report To | Format |
|-------------|----------|-----------|--------|
| Bi-weekly Pulse | Within 2 days | Team leads | Slack summary |
| Monthly | Within 1 week | Engineering leadership | Dashboard + slides |
| Quarterly | Within 2 weeks | Executive team | Full report |
